Detailed Description

Holds data necessary for an OpenGL Shading Language shader.

The shader needs to know the value for each variable, and the location for each variable Custom hashy thing? We'll put the hashy thing in the material

There's some code in here which looks for "$include" in the shader code. If it finds that, the next token is assumed to be the filename of shader library to include. Shader libraries have the extention ".glsl", have no include statements, and don't include a main() function. If a loaded shader library fails any of those conditions, the preprocessor yells at you, inserts garbage into the shader, and send that to the driver. The driver failsto compile the shader, of course, so the user knows beyond a shadow of a doubt that something is wrong.

The preprocessor log is written to "preprocessor.log" and tot he console. The shaders which are sent to the driver are found in /shader/sent. A line number is inserted at the beginning of each line for your convenience.
